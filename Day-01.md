# Day 1 - What is DevOps?

## 🔹 What I Learned
- DevOps is a combination of Development (Dev) and Operations (Ops)
- It focuses on improving collaboration between developers and IT teams
- The goal is to deliver software faster, more reliably, and continuously
- DevOps encourages automation, monitoring, and continuous feedback
- It helps reduce manual work and human errors

## 🔹 Why DevOps is Important
- Faster software delivery 
- Better collaboration between teams
- Fewer bugs and failures in production
- Quick recovery from issues
- Continuous improvement through feedback

## 🔹 DevOps Lifecycle
- Plan → Define features and requirements
- Develop → Write code
- Build → Compile and package application
- Test → Ensure quality
- Release → Prepare for deployment
- Deploy → Launch application
- Operate → Maintain system
- Monitor → Track performance and issues

## 🔹 Key DevOps Tools
- Version Control: Git
- CI/CD: GitHub Actions
- Containers: Docker
- Orchestration: Kubernetes

## 🔹 Real Use Case
In a traditional setup, developers write code and pass it to operations, which can cause delays and miscommunication.

With DevOps:
- Code is automatically built and tested
- It is deployed using CI/CD pipelines
- Teams collaborate continuously

Example:
When a developer pushes code to GitHub, a pipeline automatically builds, tests, and deploys the application.

## 🔹 Summary
DevOps is a modern approach that combines development and operations to deliver software faster, with better quality and automation. It plays a crucial role in today's fast-paced software industry.
