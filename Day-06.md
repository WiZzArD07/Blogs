# Day 6 - SDLC vs DevOps

## What I Learned
- SDLC stands for Software Development Life Cycle
- It is a traditional approach to building software in sequential phases
- DevOps is a modern approach that integrates development and operations
- DevOps focuses on continuous delivery, automation, and collaboration
- DevOps improves speed, quality, and reliability of software delivery

## SDLC Phases
- Requirement Gathering
- Design
- Development
- Testing
- Deployment
- Maintenance

## Limitations of SDLC
- Slow delivery process
- Poor communication between teams
- Late detection of bugs
- Manual deployment processes
- Difficult to handle frequent changes

## What DevOps Changes
- Breaks the barrier between development and operations teams
- Introduces automation in build, test, and deployment
- Enables continuous integration and continuous deployment (CI/CD)
- Provides faster feedback and monitoring
- Supports frequent and reliable releases

## Key Differences

| Aspect            | SDLC                          | DevOps                          |
|------------------|-------------------------------|---------------------------------|
| Approach         | Sequential                    | Continuous                      |
| Speed            | Slow                          | Fast                            |
| Collaboration    | Limited                       | High                            |
| Deployment       | Manual                        | Automated                       |
| Feedback         | Late                          | Continuous                      |
| Risk             | Higher                        | Lower                           |

## Real Use Case
In SDLC, a feature may take weeks or months to be deployed because each phase is completed separately.

In DevOps, developers push code frequently, and automated pipelines handle testing and deployment. This allows faster and more reliable releases.

Example:
A developer pushes code to GitHub, triggering a CI/CD pipeline that builds, tests, and deploys the application automatically.

## Summary
SDLC is a traditional and structured approach to software development, while DevOps is a modern methodology focused on automation, collaboration, and continuous delivery. DevOps improves efficiency and helps organizations release software faster with better quality.
