#  SOFTWARE ENGINEERING

#  UNIT 1: INTRODUCTION

##  Software Engineering
**Definition:**  
Systematic and disciplined approach to develop and maintain software.

**Key Points:**
- Uses structured process (SDLC)
- Focus on quality, cost, reliability

**SDLC Flow:**
Requirements → Design → Coding → Testing → Deployment → Maintenance

---

##  Software Characteristics
- Intangible (cannot touch)
- No wear & tear
- Highly complex
- Maintenance intensive

---

##  Software Crisis
**Causes:**
- Increasing complexity
- Poor planning
- Changing requirements

**Effects:**
- Delay
- Cost increase
- Low quality

---

##  SDLC Models

### Waterfall
- Linear and sequential
- No flexibility

### Prototype
- Based on user feedback
- Good for unclear requirements

### Spiral
- Risk-driven model
- Best for large projects

---

##  Software Quality Attributes
- Reliability
- Efficiency
- Usability
- Maintainability
- Portability

#  UNIT 3: SOFTWARE DESIGN

##  Cohesion
Degree of internal connection in module

Types:
- Functional (best)
- Sequential
- Logical

---

##  Coupling
Dependency between modules

Types:
- Data (best)
- Control
- Content (worst)

---

##  Flowchart
Graphical representation of logic

Symbols:
- Oval → Start/End
- Rectangle → Process
- Diamond → Decision

---

##  Pseudocode
Simple algorithm representation

Example:
```bash
START
INPUT n
IF n % 2 == 0
PRINT "Even"
ELSE
PRINT "Odd"
END
```
